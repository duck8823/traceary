# Security policy

[日本語](./SECURITY.ja.md)

If you believe you found a vulnerability in Traceary, please avoid opening a public issue first.

## Preferred reporting path

- Email the maintainer at `duck8823@gmail.com`
- If GitHub private vulnerability reporting is enabled for this repository, using that private report flow is also welcome

## What to include

Please include as much of the following as you can:

- affected Traceary version, tag, or commit SHA
- reproduction steps or a minimal proof of concept
- the expected impact
- whether sensitive data was exposed or persisted

## Response expectations

Traceary is maintained on a best-effort basis. The goal is to acknowledge reports within 7 days and then coordinate a fix or mitigation plan as quickly as practical.

## Supported versions

Security fixes are provided on a best-effort basis for:

- the latest tagged release
- the current `main` branch
