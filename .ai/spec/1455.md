# Issue #1455 design: verified Gemini and Antigravity usage metadata

## Structure-Behavior Design Note

### Requirement summary

- Purpose: ingest only body-free, provider-reported usage from Gemini CLI headless results and Antigravity idle status-line snapshots.
- Current state: the provider-neutral usage ledger supports additive run records, explicit unavailable call records, and immutable latest-snapshot chains, but these two host adapters are not wired.
- Expected behavior:
  - Gemini one-shot `stream-json` stores terminal `result.stats` once, split by model without also adding the aggregate total.
  - Gemini interactive `AfterAgent` records an unavailable call boundary without reading `AfterModel` request/response data.
  - Antigravity stores only idle cumulative input/output as a non-additive session snapshot and records each observable Stop boundary as unavailable.
  - redelivery is idempotent; conflicting duplicate or regressing metadata fails closed.
- Non-goals: infer tokens, persist prompts/responses/current usage/quota/email, scrape billing APIs, install Gemini `AfterModel`, or auto-replace an operator's Antigravity status-line renderer.

### Conceptual model

| Concept | State | Behavior | Constraint / invariant |
|---|---|---|---|
| Gemini terminal run | terminal status, timestamp, per-model counters | becomes additive run observations | one terminal result; model rows sum to the aggregate; aggregate is not stored again |
| Gemini interactive boundary | session and durable hook delivery | becomes unavailable call evidence | no token or model value is inferred |
| Antigravity conversation snapshot | conversation, model, cumulative input/output, revision | supersedes the previous snapshot | only idle payloads; counters never regress; latest revision alone is aggregate-eligible |
| Antigravity Stop boundary | conversation and resolved transcript turn boundary | becomes unavailable call evidence | identity describes a hook boundary, not a provider request |

### Responsibility assignment

| Responsibility | Owner | Reason to change | Not owner / reason |
|---|---|---|---|
| provider-neutral usage invariants | existing domain usage observation model | accounting and snapshot-chain rules | CLI must not duplicate these rules |
| Gemini/Antigravity capture decisions | host-specific application use cases | source classification and observation mapping | filesystem adapters must not write SQLite |
| JSON allowlist parsing and stream bounds | filesystem adapters | host wire format changes | domain/application must not depend on host DTOs |
| process/hook wiring and body-free spooling | CLI presentation | command and hook protocol changes | use cases must not parse Cobra or environment state |
| idempotent persistence and snapshot-head lookup | SQLite datasource | transaction and query behavior changes | use case must not know SQL/schema |

### Boundaries / interfaces

| Boundary or interface | Consumer | Hidden detail | Error contract |
|---|---|---|---|
| `GeminiHeadlessUsageStream` | session runner | JSONL chunks and ignored message/tool bodies | malformed/conflicting/oversized terminal metadata returns no partial samples |
| `GeminiUsageCaptureUsecase` | session runner and Gemini hook | domain construction and unavailable fallback | invalid source/counters fail closed; redelivery is counted as already applied |
| `AntigravityUsageCaptureUsecase` | status-line and Stop handlers | snapshot head/revision and observation identity | regression/conflicting chain is rejected; identical snapshot is idempotent |
| `FindSnapshotHead` repository port | Antigravity use case | SQL head query | zero or one head; multiple heads are a conflict |

### Behavior tests

| Behavior | Given | When | Then | Level |
|---|---|---|---|---|
| Gemini per-model capture | valid v0.46.0 terminal fixture | stream completes | one additive run record per model, no aggregate duplicate | adapter/use case |
| Gemini known zero | terminal stats contains zero | capture runs | zero is known, not unavailable | adapter/use case |
| Gemini parse failure | valid terminal followed by malformed/conflicting data | stream completes | mode retained, samples discarded, unavailable run fallback recorded | adapter/CLI |
| Gemini interactive privacy | AfterAgent contains private response | usage hook spools | only session/boundary identifiers survive; unavailable counters recorded | CLI |
| Antigravity idle snapshot | idle body-free status payload | capture runs twice | exact redelivery is idempotent | adapter/use case |
| Antigravity supersession | larger cumulative counters arrive | capture runs | revision increments and supersedes previous head | use case/SQLite |
| Antigravity regression | either cumulative counter decreases | capture runs | no new snapshot is persisted | use case |
| Antigravity privacy | status payload contains email/quota/current usage | capture runs | none of those values enters identity, errors, or persistence | adapter |
| Antigravity Stop unavailable | completed transcript turn is resolved | Stop runs twice | one unavailable call boundary is retained idempotently | CLI/use case |

### Risks / rollback

- Procedural-implementation risk: avoid placing snapshot-chain decisions in the status-line Cobra handler.
- Premature-abstraction risk: Gemini and Antigravity retain separate types/use cases because additive terminal runs and cumulative snapshots have different invariants.
- Migration/compatibility: no schema migration; existing provider-neutral v29 tables and bundle format are reused additively.
- Rollback trigger: any duplicate aggregate, counter regression accepted as a successor, raw body/account/quota value in persisted/spooled metadata, or loss of normal child stdout.
- Rollback: remove the new hook/stream wiring while retaining the already-compatible ledger rows; no existing row or schema must be rewritten.

## Implementation checkpoint

The implementation will use two host-specific application boundaries, a bounded Gemini stream adapter, an allowlisted Antigravity status payload decoder, and the existing immutable snapshot domain model. It will not modify schema or install `AfterModel`.
