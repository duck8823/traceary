## Field registry

### Canonical order（built-in default）

現行 compact 出力と byte-for-byte 互換を守るため、built-in default は変更しない:

```
ts, kind, session, ws, message
```

### 対応 field 名と source

| Field | Description | Event source | Default |
|---|---|---|---|
| `ts` | `HH:MM:SS`（`--utc` で UTC） | `Event.CreatedAt()` | ✅ |
| `kind` | event kind 生文字列 | `Event.Kind()` | ✅ |
| `session` | `sess=<first-8>` | `Event.SessionID()` | ✅ |
| `ws` | `ws=<basename>` | `Event.Workspace()` | ✅ |
| `client` | recording channel (空は `-`) | `Event.Client()` | ❌ |
| `agent` | actor | `Event.Agent()` | ❌ |
| `message` | body（whitespace normalize + truncate） | `Event.Body()` | ✅ |
| `exit_code` | command audit の exit code（非 audit や未記録は `-`） | `command_audits.exit_code` (lazy hydrate) | ❌ |
| `id` | full event ID（lossy shortening なし） | `Event.EventID()` | ❌ |

### Precedence

`explicit --fields` > `config.json` > built-in default

`cmd.Flags().Changed("fields")` で explicit flag と config fallback を区別する。

### Config schema

```json
{
  "redact": {
    "extra_patterns": ["..."]
  },
  "read": {
    "columns": ["ts", "kind", "session", "ws", "message"]
  }
}
```

後方互換: 既存の `redact` だけを持つ config.json はそのまま有効。`read` / `read.columns` がない場合は built-in default にフォールバック。

## 排他 / 互換ルール

- `--fields` と `--wide` 併用は **conflict error**。`--wide` 単体時は legacy 7-column 維持（config default ありでも 7-column 出力）。
- `--fields` と `--json` は **併用可**（ただし `--json` 側は unaffected で structured schema 維持）
- `--fields` 内の unknown field / empty token / duplicate field は **clear error**（supported list 付き）
- config.json の `read.columns` 内に unknown field / duplicate / empty が含まれていれば affected command で clear error（silent ignore しない、fail-closed）

## 実装手順

### Step 1: config loader 拡張
- `presentation/config.go` に `Config{Redact, Read}` を導入
- `LoadConfig()` 相当の汎用 loader を追加（既存 `LoadExtraRedactPatterns()` は薄いラッパーで保持、後方互換）
- `main.go` で 1 回読み込んで `RootCLI` に両方注入（`WithDefaultReadColumns` 等の option）

### Step 2: Field registry 新設
- `presentation/cli/read_fields.go` に canonical field ID 型、allowlist、precedence resolver、validation 関数
- `presentation/cli/read_fields_test.go` で precedence / unknown / empty / duplicate / supported-list error を網羅

### Step 3: `--fields` flag の plumbing
- `input.go` に共通 flag 追加 helper（StringSliceVar）
- `list.go` / `search.go` / `tail.go` で flag を受け取って resolver に渡す

### Step 4: formatter を field-driven に
- `event_text_formatter.go` で compact row を token list から組み立てる loop 設計に書き換え
- built-in default の出力が現行と byte-for-byte 同じになることを test で固定
- 新 field (`client`, `agent`, `exit_code`, `id`) の render を追加
- `event_output.go` の writer が field list を受け取るように変更

### Step 5: `exit_code` の lazy hydrate
- `exit_code` が選択された時だけ `EventUsecase.Show()` で `command_audit.exit_code` を引く helper を `list.go` / `search.go` / `tail.go` の出力直前に追加
- application / queryservice / SQLite query surface は変更しない
- non-audit event や exit code 未記録は `-`

### Step 6: Doctor 調整
- `doctor.go` の config health message を「`extra redaction patterns` のみ影響」表現から「config-backed features」表現に修正

### Step 7: Docs 更新
- `docs/cli/README.md` + `README.ja.md`: `--fields` 説明、precedence、`--wide` との conflict、supported field list
- `docs/environment/README.md` + `README.ja.md`: `read.columns` の config 例、unknown field behavior

## 注意事項

### セキュリティ

- field 名は reflection や自由文字列ではなく closed allowlist で解決（内部 field や audit payload 本文の誤露出を防ぐ）
- `--json` は既存 structured schema 維持（machine-readable surface を壊さない）
- `exit_code` hydrate でも表示対象は exit code のみ（`input`/`output` は流用しない）
- config.json の unknown field は fail-closed で typo を silent ignore しない
