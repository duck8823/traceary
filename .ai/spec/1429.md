# Structure-Behavior Design Note: workspace identity diagnostics and duplicate measurement

## Requirement summary

- Purpose: make workspace-attribution quality, stable-ID redeliveries, and historical heuristic duplicate candidates measurable without changing canonical session/event provenance.
- Current behavior: v0.30 ingestion records delivery ledger rows and workspace observations, while the existing reversible content-event dedupe command reports historical candidates only as groups.
- Expected behavior: expose a body-free report by host/source hook, record every stable-ID delivery attempt so exact-redelivery rate is measurable, support explicit reviewed workspace aliases, and retain the existing dry-run-by-default reversible cleanup path.
- Non-goals: automatic alias inference, body-only identity, automatic historical deletion, default filter widening, session/event workspace rewrites, or reporting raw event bodies.

The normative contract is `docs/operations/workspace-identity-contract.md`.

## Conceptual model

| Concept | State | Behavior | Constraint / invariant |
|---|---|---|---|
| Delivery attempt | delivery record ID, attempted event ID, outcome, observed time | records accepted, conflict, or exact-redelivery observation | append-only; one row per repository attempt identity; no event body |
| Workspace identity report | coverage, relationship counts, delivery rates, heuristic rates, conflict samples | aggregates by source client and hook | read-only; exact and heuristic metrics remain separate |
| Reviewed alias | session ID, alias workspace, reviewer, time, note | reclassifies current report view from conflict to explicit alias | never rewrites session, event, or stored observation |
| Historical candidate source statistic | scanned rows, groups, candidate rows | measures reversible content-equality candidates | heuristic only; never presented as proven redelivery |
| Conflict sample | event/session/source identifiers | points operators to body-free review targets | no prompt, transcript, command input, or output |

## Responsibility assignment

| Responsibility | Owner | Reason to change | Not owner / reason |
|---|---|---|---|
| delivery-attempt atomic write | SQLite event-delivery repository | transaction/schema ordering changes | usecase must not own SQL or retries |
| relationship and rate projection | workspace identity query service + SQLite adapter | report query/aggregation changes | CLI must not reconstruct provenance joins |
| alias validation and orchestration | workspace identity usecase/domain alias model | review operation rules change | SQLite must not decide whether reviewer/session/workspace input is valid |
| historical candidate grouping | existing store dedupe datasource | heuristic grouping rules change | report must not duplicate the cleanup algorithm |
| command flags and JSON/text rendering | CLI | operator contract and localization change | usecase must not own presentation DTOs |

## Boundaries / interfaces

| Boundary or interface | Consumer | Hidden detail | Error contract |
|---|---|---|---|
| `WorkspaceIdentityUsecase.Report` | `report workspace-identity` | SQL joins, current alias reclassification | schema/read failure fails the report; no writes |
| alias add/remove/list | `store workspace-alias` | alias table and FK behavior | invalid input fails before persistence; missing removal is an idempotent no-op |
| delivery-attempt insert | unified event delivery transaction | migration availability and attempt outcome | attempt and logical event decision commit or roll back together |
| `DedupeContentEvents` source stats | dedupe CLI and identity report | proximity grouping and canonical row choice | dry-run remains non-mutating; malformed groups remain skipped |

## Behavior tests

| Behavior | Given | When | Then | Level |
|---|---|---|---|---|
| exact rate | one accepted delivery and one exact retry | report runs | attempts=2, exact=1, exact rate=0.5 |
| conflict rate | reused native ID with changed semantics | report runs | accepted/conflict attempts are separated from exact retries |
| atomic attempt | boundary callback fails | transaction rolls back | no event, delivery, observation, or attempt row leaks |
| relationship report | exact/descendant/conflict/unknown observations | report runs | counts and rates are grouped by client/source hook |
| reviewed alias | conflict exists | alias is explicitly added | stored relationship remains conflict; current report shows explicit alias |
| alias reversal | reviewed alias exists | alias is removed | current report returns to conflict without provenance rewrite |
| coverage | events include one missing primary observation | report runs | event count, covered count, missing count, and coverage rate agree |
| heuristic separation | near-simultaneous equal content exists | report runs | heuristic candidate rate is reported separately from exact delivery rate |
| privacy | conflict sample points to an event with a sensitive body | JSON/text renders | body and audit payload are absent |

## Risks / rollback

- Procedural risk: calculating the same historical candidate groups in both dedupe and report. The report consumes source statistics returned by the existing dedupe dry-run.
- Premature abstraction risk: do not create a generic analytics framework; the query and alias interfaces stay workspace-identity-specific.
- Migration/compatibility: migration 23 adds only `hook_delivery_attempts`; older binaries ignore it, and current event/session/filter columns are unchanged.
- Privacy: the attempt table and reports contain hashes/IDs/source facts only; conflict samples never contain bodies.
- Rollback: deploy the previous binary and retain the additive table/alias rows. No down migration or data rewrite is required.
- Rollback trigger: event/session row counts change, default filter membership changes, report exposes body content, runtime ingest is blocked, or copied-store dogfood reports unexplained known conflicts above 5%.
