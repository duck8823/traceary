# Issue #1498: diagnose active Codex capture gaps

## Structure-Behavior Design Note

### Requirement summary

- **Goal:** make `traceary doctor --client codex` distinguish committed capture,
  durable delivery backlog, an exercised Stop with missing usage, and no recent
  evidence for the canonical workspace.
- **Current state:** doctor reports hook trust, config, plugin version, and the
  global spool backlog separately. A configured Codex route with zero committed
  events can therefore look unrelated to thousands of pending Codex deliveries.
- **Expected behavior:** one body-free diagnostic correlates recent event
  metadata, finalized usage aggregates, and allowlisted spool metadata for the
  same canonical workspace, then reports every lifecycle boundary as
  `stored`, `delivery_pending`, `stored_and_delivery_pending`, or
  `not_observed`.
- **Non-goals:** infer token counts, inspect or render prompt/transcript/tool
  bodies, claim that an unexercised hook failed, drain the spool automatically,
  or change provider accounting.

### Conceptual model

| Concept | State | Behavior | Constraint / invariant |
|---|---|---|---|
| Codex surface | plugin-managed or manual hooks, plugin key/trust, Traceary version | identifies the capture route being diagnosed | an undetectable route is never reported as trusted |
| Canonical workspace | repository identity resolved from `--project-dir` | scopes stored and pending evidence | local aliases resolve through the same repository detector as event coverage |
| Capture boundary | session start, prompt, tool, compact, stop, usage | reports stored, delivery pending, or not observed | absence alone does not become a fabricated failure |
| Pending delivery | spool command/action plus allowlisted session/cwd metadata | proves the host reached Traceary but the write has not committed | private payload fields never enter the diagnostic result |
| Usage evidence | finalized Codex aggregate rows for the workspace | distinguishes known, explicitly unavailable, pending, and missing-after-stop | unavailable is never numeric zero or `unknown` |

### Responsibility assignment

| Responsibility | Owner | Reason to change | Not owner / reason |
|---|---|---|---|
| Read event metadata and shared report snapshot | doctor CLI orchestration | operational diagnostic composition | storage adapters already expose body-free projections |
| Map evidence to boundary states and reason codes | pure Codex doctor classifier | diagnostic policy changes | Cobra/report rendering must not duplicate decisions |
| Canonical workspace resolution | existing doctor workspace resolver | repository alias semantics | spool parser must not invent workspace identities |
| Usage accounting and exactly-once persistence | existing Codex usage usecase/repository | provider accounting changes | diagnostic remains read-only |
| Hook trust and plugin version checks | existing Codex doctor checks | host/plugin contract changes | capture classifier consumes their result only as surface identity |

### Boundaries / interfaces

| Boundary or interface | Consumer | Hidden detail | Error contract |
|---|---|---|---|
| `EventMetadataUsecase.List` | Codex capture diagnostic | event bodies and SQLite layout | read failure makes the diagnostic fail closed |
| `ReportUsecase.Generate` | Codex capture diagnostic | usage row selection and aggregate rules | read/validation failure makes usage diagnosis unavailable |
| spool metadata projection | classifier | prompt, transcript, command, and tool-output bodies | malformed payload becomes unscoped evidence and is never echoed |
| `doctorCheck` | text/JSON doctor output | classifier structs | emits a stable reason and explicit boundary states |

### Behavior tests

| Behavior | Given | When | Then | Level |
|---|---|---|---|---|
| Current backlog detected | managed hooks, zero stored events, Codex spool records for the workspace | doctor runs | WARN with `hook_spool_backlog`, pending boundaries, surface, version, client, and canonical workspace |
| Privacy | spool payload contains private prompt/tool/transcript fields | diagnostic projects it | output contains only command/action/session/cwd-derived evidence and no private body |
| Stored coverage | recent body-free metadata covers session/prompt/tool/compact/stop and usage exists | doctor runs | PASS with stored boundaries and known or explicitly unavailable usage |
| Missing usage | a stored Stop exists without finalized usage or pending usage delivery | doctor runs | WARN with `usage_missing_after_stop` |
| No evidence | configured route has no recent stored or pending evidence | doctor runs | WARN with `no_recent_capture_evidence`, not a pass |
| Alias consistency | project path and spool cwd are aliases of one repository | evidence is correlated | both resolve to one canonical workspace |
| Existing accounting | duplicate Codex terminal delivery | existing capture usecase/repository reconcile it | one additive winner remains; diagnostic only reads the result |

### TDD plan

| Behavior | Red | Green | Refactor target |
|---|---|---|---|
| Boundary classifier | table tests fail for backlog/missing usage/no evidence/healthy | implement pure reason/state classifier | keep formatting separate from decisions |

| Documentation | operators cannot interpret reason codes | add EN/JA Codex integration guidance | keep command examples paired |

### Risks / rollback

- **Procedural risk:** do not move accounting rules into doctor. It only
  classifies durable evidence already produced by the usage pipeline.
- **Premature abstraction:** implement a Codex-specific diagnostic because hook
  boundary and usage contracts differ by host; do not introduce a universal
  capture-health framework without a second proven consumer.
- **Migration / compatibility:** no schema or hook contract change. The new
  doctor check is additive and read-only.
- **Performance:** reuse the global spool scan, read at most 500 recent event
  metadata rows, and cap the shared report snapshot used for usage evidence.
- **Rollback trigger:** private body leakage, false PASS with a backlog or
  stored Stop/missing usage, canonical-workspace divergence, or material doctor
  latency regression.
- **Rollback:** remove the additive check and documentation; no durable data
  rollback is required.
- **PR plan:** one issue/one PR. Commit the classifier/integration/tests first,
  then paired documentation.

### Design checkpoint

Proceed with a read-only Codex-specific doctor check. Reuse the canonical
workspace resolver, body-free event/report projections, and the already-required
spool scan. Treat backlog and stored-Stop-without-usage as actionable warnings;
report no evidence as an explicit warning; never infer token values or inspect
private bodies.
