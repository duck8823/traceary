## Structure-Behavior Design Note

### Requirement summary
- Purpose: persist only the body-free `usage.record` rows proven in Kimi Code's local session wire, while making uncorrelated lifecycle completions explicitly unavailable.
- Current state: Kimi lifecycle and transcript hooks are wired, and the sanitized v0.27.0 fixture proves the main-agent wire shape; no usage adapter consumes it.
- Expected behavior: scan a bounded, contained wire path; preserve counter presence and known zero; record each source row idempotently as excluded partial evidence; record Stop and SessionEnd availability separately.
- Out of scope: estimating retry/cardinality, promoting Kimi records into aggregates, parsing prompt/content/tool bodies, provider pricing, and treating compact token counts as provider usage.

### Conceptual model
| Concept | State | Behavior | Constraint / invariant |
| --- | --- | --- | --- |
| Kimi source record | host session, agent, usage ordinal, model, counters, timestamp | becomes one finalized excluded observation | identity is independent of file path and Traceary delivery; changed metadata under the same identity conflicts |
| Lifecycle boundary | Stop or SessionEnd plus stable wire turn ordinal when present | becomes an unavailable call observation | never claims correlation to a source record |
| Counter presence | known integer or absent | preserves known zero and unavailable dimensions | negative/fractional values fail closed; totals are not inferred |
| Wire source | session index and contained `agents/main/wire.jsonl` | exposes only body-free usage metadata | non-usage rows decode only their discriminator; symlink/path escape is rejected |

### Responsibility assignment
| Responsibility | Owner | Reason to change | Not owner / reason |
| --- | --- | --- | --- |
| Session index containment and JSONL parsing | `infrastructure/filesystem.KimiUsageSource` | Kimi on-disk contract changes | CLI/usecase must not know file layout or host DTOs |
| Provider-neutral observation mapping and replay semantics | `application/usecase.KimiUsageCaptureUsecase` | accounting policy changes | filesystem adapter must not construct domain observations |
| Hook orchestration and store initialization | Kimi CLI adapter | hook protocol changes | it must not parse usage records or decide counters |
| Observation identity/conflict persistence | usage domain and SQLite repository | persistence invariant changes | Kimi code reuses the shared contract |

### Boundaries / interfaces
| Boundary or interface | Consumer | Hidden detail | Error contract |
| --- | --- | --- | --- |
| `KimiUsageSource.Load` | Kimi capture usecase | index/wire paths, symlinks, JSON DTOs, size guards | missing source is an empty result; malformed authoritative usage fails without body/path leakage |
| `KimiUsageCaptureUsecase.Capture` | Stop/SessionEnd hook adapter | observation construction and replay counting | identical replay is already-applied; changed counters conflict |
| `WithKimiUsage` | composition root | concrete source/repository | missing dependency is a configuration error |

### Behavior tests
| Behavior | Given | When | Then | Level |
| --- | --- | --- | --- | --- |
| Private-free scan | wire contains prompt/content/tool and two usage rows | load | only two usage samples and the turn ordinal are returned | infrastructure |
| Presence preservation | cache counters are absent or zero | capture | absent is unavailable and zero is known zero | application |
| Durable idempotency | same wire is scanned by repeated Stop/SessionEnd delivery | record | source records do not double count; boundaries are stable | integration/application |
| Conflict detection | same source ordinal carries changed counters | record | capture fails closed with the typed conflict | application |
| Unsupported completion | no correlatable usage row exists | Stop/SessionEnd | excluded unavailable call evidence is recorded | CLI/application |
| Path safety | index points outside the Kimi sessions root | load | source is rejected without reading the target | infrastructure |

### Risks / rollback
- Procedural risk: do not extend transcript extraction to return usage; usage has a separate body-free filesystem port.
- Premature abstraction risk: keep Kimi-specific DTOs and semantics separate until another host shares the exact wire contract.
- Migration / compatibility: no schema change; all Kimi records use existing excluded observations and are ignored by additive aggregates.
- Rollback trigger: disable `WithKimiUsage` wiring and remove capture calls if Kimi's versioned wire contract changes; lifecycle/transcript capture remains intact.
