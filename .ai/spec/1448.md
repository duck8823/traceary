# Issue #1448: host usage sources and privacy boundaries

## Requirements

- Classify Codex, Claude Code, Gemini CLI, Antigravity CLI, Grok Build, and Kimi Code usage sources as available, partial, or unavailable from official contracts and sanitized local probes.
- Identify the terminal event and retry/stream deduplication rule for every available path.
- Keep prompt bodies, response bodies, reasoning text, tool payloads, credentials, and network telemetry outside the usage model.
- Preserve the v0.26 decision that Traceary does not export GenAI telemetry over the network.
- Turn the findings into exact implementation boundaries for #1451, #1447, #1455, #1450, and #1452.

## Conceptual model

| Concept | Responsibility |
|---|---|
| Usage source | A versioned local host contract that carries provider-reported counters. |
| Usage observation | Immutable counters plus host, model, session/call identity, source version, and availability. |
| Observation scope | `call`, `run`, or non-additive `session_snapshot`; a source adapter selects exactly one accounting scope for a path. |
| Terminal authority | The event after which an observation may be finalized exactly once. |
| Availability | `available`, `partial`, or `unavailable`; zero is a numeric value, never a missing-data marker. |
| Privacy boundary | A selective metadata reader that never persists or emits private body fields. |

## Responsibility and interface boundaries

- Host adapters translate only documented or live-verified metadata fields. They do not estimate missing provider counters.
- A host path selects one accounting representation. Summary and child-call observations from the same run are never both aggregate-eligible.
- The provider-neutral model (#1456) owns availability and idempotency. Adapters do not encode missing values as zero.
- Line-oriented readers may inspect a mixed local JSONL file only to recognize an event discriminator and decode a usage event. Non-usage lines must not be decoded into domain objects, logged, or persisted.
- Terminal correlation belongs to each adapter because the authoritative boundary differs by host. Persistence and aggregation consume already-finalized observations.
- Network collectors and OTel exporters are not part of these interfaces.

## Observable behavior

1. A supported terminal event produces one usage observation per authoritative call or run, or one superseding non-additive session snapshot.
2. Replayed, duplicated, or intermediate stream events do not increase totals.
3. A failed or aborted path without provider-reported counters records `unavailable`, not zero.
4. A partial source exposes only the claims its contract supports and labels the rest unavailable.
5. Metadata extraction never places prompts, responses, thinking, tool arguments/results, email addresses, credentials, or transcript bodies in Traceary usage records.

## Behavior tests for implementation issues

- Replay the same terminal event twice and observe one persisted call.
- Feed both a run summary and its child-call rows and observe only the adapter-selected representation in aggregates.
- Feed intermediate cumulative snapshots followed by a terminal boundary and observe only the terminal delta.
- Feed an error/abort without usage and observe `unavailable` with no numeric counters.
- Feed a genuine zero-valued terminal usage record and preserve known zero.
- Include private body fields beside usage metadata and verify they cannot appear in the stored model, CLI JSON, logs, or MCP output.
- Reject counter regression, ambiguous terminal correlation, duplicate call identities with conflicting counters, and undocumented fields.

3. #1451, #1447, #1455, #1450, and #1452 add one host adapter per issue using the source matrix in the research document.

## Migration and rollback

- #1448 changes documentation only. It creates no database or runtime compatibility risk.
- Later migrations must be additive; older rows remain readable as `unavailable` rather than fabricated zero usage.
- Every adapter can be disabled independently without deleting existing observations.
- Session snapshots remain immutable: a newer snapshot supersedes an older one, and queries select the latest snapshot instead of mutating or adding the history.
- If a host contract changes or terminal correlation becomes ambiguous, fail closed to `unavailable` and retain source-version evidence for repair.

## Design checkpoint and self-review

- The design keeps provider parsing out of CLI handlers and aggregate query services.
- Decision logic (availability, idempotency, terminal finalization) is separated from filesystem I/O.
- No boolean “has usage” flag collapses unknown, unavailable, zero, and known-positive states.
- Tests target persisted/queried behavior rather than private parser call order.
- Release remains blocked until #1457 proves reconciliation and privacy behavior; rollback is adapter disablement, not destructive data rewrite.
