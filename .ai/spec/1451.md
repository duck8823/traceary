# Issue #1451: capture verified Codex usage metadata

## Structure-Behavior Design Note

### Requirement summary

- Purpose: persist authoritative Codex token usage from local, host-owned JSONL without storing or parsing prompt/response bodies for accounting.
- Current state: the provider-neutral usage aggregate and SQLite repository exist, but Codex Stop hooks only record transcript/session boundaries.
- Expected behavior: every verified Codex usage record becomes one idempotent additive call observation; duplicate Stop delivery replays without double counting; a stable Stop delivery with no readable usage becomes one excluded, terminal observation whose counters are explicitly unavailable.
- Non-goals: provider billing estimates, body-length inference, arbitrary Codex event persistence, request retry reconstruction without host evidence, aggregate CLI/MCP reporting, or other host adapters.

### Conceptual model

| Concept | State | Behavior | Constraint / invariant |
|---|---|---|---|
| Codex usage source | local rollout/exec JSONL | returns body-free usage samples | only `token_count.info.last_token_usage` and `turn.completed.usage` are authoritative; unrelated JSON fields are ignored |
| Usage sample | source identity, model, timestamp, counters | becomes a finalized additive call observation | observation ID is deterministic from session, source file, and source line; absent dimensions remain unavailable, not zero |
| Missing-usage boundary | session and host-native Stop `event_id` | becomes one finalized excluded observation | no body-derived identity; without stable delivery identity the adapter skips rather than fabricates one |
| Replay | same observation identity and semantics | returns already-applied | differing semantics fail through the usage aggregate/repository conflict contract |

### Responsibility assignment

| Responsibility | Owner | Reason to change | Not owner / reason |
|---|---|---|---|
| Locate and parse Codex JSONL | `infrastructure/filesystem` Codex source | Codex file layout/schema or IO guard changes | CLI/usecase must not know filesystem traversal or raw JSON envelopes |
| Normalize samples and unavailable boundaries into domain observations | Codex usage application usecase | adapter mapping/terminal availability changes | filesystem must not own persistence; CLI must not own accounting rules |
| Enforce idempotency, terminal states, and storage | existing usage domain + repository | provider-neutral invariant/schema changes | Codex adapter must not reimplement reconciliation |
| Resolve hook payload/session/delivery and initialize DB | presentation CLI | hook protocol/composition changes | usecase must not depend on Cobra or hook JSON |
| Register Stop usage capture | Codex hook handler/plugin manifest | packaged hook contract changes | parser must not mutate host configuration |

### Boundaries / interfaces

| Boundary or interface | Consumer | Hidden detail | Error contract |
|---|---|---|---|
| `CodexUsageSource.Load` | Codex usage usecase | `CODEX_HOME`, dated rollout paths, JSONL envelopes, size/line guards | missing source is a successful empty result; unsafe path/file or malformed authoritative usage fails closed |
| `CodexUsageCaptureUsecase.Capture` | hook CLI | source parsing, observation construction, replay handling | returns applied/already-applied counts; repository/source failures are wrapped without private payloads |
| `traceary hook usage codex` | packaged Stop hook | DB wiring and source adapter | unsupported clients are rejected; missing stable delivery ID is a safe no-op when usage is absent |

### Behavior tests

| Behavior | Given | When | Then | Level |
|---|---|---|---|---|
| rollout capture | private-data-free `token_count` fixtures with known zero/cache/reasoning | source loads and usecase captures | exact counters and model/version are stored once | filesystem + usecase + SQLite integration |
| direct exec capture | `turn.completed.usage` fixture with partial fields | capture | reported values are known and absent dimensions unavailable | filesystem/usecase |
| duplicate delivery | same source and Stop payload twice | capture twice | second pass is already-applied and totals do not duplicate | usecase/SQLite |
| compaction/resume | completed-call token records plus a zero-dimensional cumulative compaction carryover | capture | completed calls keep deterministic identity and the carryover is excluded from additive accounting | filesystem |
| aborted/missing usage | stable Stop event without usage source | capture twice | one excluded finalized observation with unavailable counters | usecase/SQLite |
| privacy | fixtures include prompt/response/tool fields beside usage | capture | observations contain only IDs, source/model, counters, cost state, and terminal metadata | filesystem/usecase |
| unsafe source | oversized, symlinked, or escaping source path | load | reject without reading arbitrary files | filesystem |

### TDD plan

| Behavior | Red | Green | Refactor target |
|---|---|---|---|
| parse verified Codex variants | fixture tests fail because no source exists | bounded parser yields typed samples | share numeric optional decoding without generic host abstraction |
| map availability and replay | fake repository tests fail | usecase builds existing usage aggregate and handles stable missing boundary | keep provider decisions in adapter mapper |
| persist from hook | command/integration test fails | inject source/repository usecase and register Stop command | CLI remains protocol orchestration only |

### Risks / rollback

- Procedural risk: avoid putting JSONL traversal and token semantics into `runHookUsage`; the CLI only resolves payload and delegates.
- Premature abstraction risk: implement the Codex source and capture port only. Other provider issues may reuse the generic hook command but do not share an invented universal raw DTO.
- Compatibility: no migration is required; observations use schema 27/28 tables. Existing hooks remain valid until upgraded, while upgraded Stop hooks add one body-free capture command.
- Performance guard: source files are regular, non-symlink files with a bounded maximum size and scanner line size; session matching uses a fixed-depth filename glob rather than an unbounded home scan.
- Rollback: remove the Stop usage command and Codex adapter wiring. Persisted provider-neutral observations remain valid and can be retained; no schema downgrade is needed.

Design checkpoint: adopt a local filesystem source plus a Codex-specific application mapper. Record per-call `last_token_usage`/`turn.completed.usage` as additive observations and stable missing Stop boundaries as excluded/unavailable observations. Never infer usage from transcript bodies, command counts, or cumulative deltas.
