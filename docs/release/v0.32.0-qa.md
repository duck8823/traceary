## Scope and privacy boundary

The release candidate was exercised against the local production-shaped store over an eight-day half-open interval, plus isolated body-free headless probes. The report used `--result-cap 0 --page-size 100000`, so every source extent was complete rather than a sampled prefix. QA inspected only session/event counts, availability states, provider/model identifiers, terminal classifications, and numeric usage counters. No prompt, response, thinking, tool body, credential, or raw transcript was required.

The requested date interval `2026-07-17` through `2026-07-24` in `Asia/Tokyo` resolved to `[2026-07-16T15:00:00Z, 2026-07-24T15:00:00Z)`.

## Follow-up findings and disposition

1. #1498 / PR #1503 fixed Codex completion evidence that was incorrectly reported as absent while its durable delivery was still queued.
2. #1504 / PR #1505 fixed unbounded/retry-first spool replay. Current deliveries are processed first, retry work is bounded and fair, concurrent consumers use isolated ownership, and stale claims recover safely.
3. #1506 / PR #1507 fixed Grok headless observations being timestamped at the Unix epoch. The merged candidate assigns UTC ingestion time while portable replay equality deliberately ignores local observation/finalization time.

All three follow-ups were merged after deterministic validation and live body-free dogfood. The required independent reviewer topology was unavailable for PRs #1503, #1505, and #1507; the maintainer explicitly accepted that residual risk for each merge.

## Operational observations

The final `doctor --fix --dry-run` found 8,678 readable pending spool records, zero unreadable records, and a 3.2 GiB SQLite store. The currently installed v0.31 binary still reports the Traceary workspace as `hook_spool_backlog` with no committed usage observation. This is expected pre-upgrade operational debt, not an unfixed v0.32 code path: the v0.32 candidate was separately verified to process the current event before a bounded retry batch and to reduce the backlog. Full `doctor --fix` was intentionally not applied because it would also perform unrelated session/config maintenance.

Post-release verification must confirm that the installed v0.32 binary drains the backlog and that new Codex completions no longer increase it. Store compaction remains an operator action (`traceary store gc --dry-run`) and is not required to alter usage-accounting semantics.

## Release decision

Pass. All release-blocking findings discovered by QA were filed, implemented, merged, and re-verified. Known usage-capable live probes reached 100% availability after the fixes; unavailable and partial host paths are explicitly classified. No duplicate accounting or private-body dependency remains. The historical spool backlog is a monitored post-upgrade operational risk, not a release-blocking correctness defect.
