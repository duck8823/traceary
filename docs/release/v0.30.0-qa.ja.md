# v0.30.0 release QA / dogfood

[English](./v0.30.0-qa.md)

## 対象

機能ごとの copied-store・並行実行 dogfood に加えて、使い捨ての新規 SQLite store で release candidate を検証しました。最終確認の対象は、本文なしの read、workspace/delivery identity 診断、one-shot lifecycle の終了結果、完全・部分 report、日付境界、従来の wide 表示、不正な resource 制御値です。

## 新規 store での最終確認

最終 implementation head から candidate binary を作り、`/private/tmp` 配下の store を使いました。

- `session run` は success（exit 0）、failure（exit 7）、timeout（exit 124）を、terminal reason `success` / `failure` / `timeout` の終了済み one-shot session として記録しました。
- `session list --json` は 3 session を返し、それぞれに lifecycle event が 2 件ありました。
- `list --wide` は従来形式の 8 field header を出力し、`list --fields ... --json` は要求した projection だけを返しました。
- `--page-size 2` の完全 report は 3 session・6 event を全件走査しました。`--result-cap 1` の部分 report は `coverage=partial`、source ごとの観測範囲、`response_truncated=true`、`truncation_reason=result_cap` を返し、不完全な割合は提示しませんでした。
- 日付のみの `--from 2026-07-22 --to 2026-07-22 --timezone UTC` は半開区間 `[2026-07-22T00:00:00Z, 2026-07-23T00:00:00Z)` に解決されました。
- `report workspace-identity --json` は新規 store の observation coverage が完全で、conflict と重複 delivery attempt がないことを報告しました。
- `--page-size 9223372036854775807` は `page size must be less than or equal to 100000` で exit 1 となり、panic と runtime stack は出ませんでした。

## Follow-up の発見と対応

release 全体確認で、巨大な report page size により slice の事前割り当てが panic する問題を発見しました。follow-up Issue [#1479](https://github.com/duck8823/traceary/issues/1479) を作成し、PR [#1480](https://github.com/duck8823/traceary/pull/1480) で実装しました。blocking なテスト不足の指摘を修正して独立再レビューを通し、local validation と CI の成功後にマージしています。修正済み candidate で同じ dogfood command を繰り返し、上限付きの通常 validation error になることを確認しました。

## Release 判定

合格。release-blocking な QA finding はすべて Issue 化・修正・検証済みです。v0.30.0 を止める data loss、互換性、security、availability の未解決問題はありません。
