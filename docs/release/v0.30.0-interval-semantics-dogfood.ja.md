# v0.30.0 要求期間・実効期間のドッグフーディング

[English](./v0.30.0-interval-semantics-dogfood.md)

## 対象

Issue #1432 を、ローカルでビルドしたバイナリと使い捨ての新規 SQLite
ストアで確認しました。稼働中の Traceary ストアは変更していません。

## 実行コマンド

```sh
go build -o /private/tmp/traceary-issue-1432 .
/private/tmp/traceary-issue-1432 report \
  --db-path /private/tmp/traceary-issue-1432-qa.db \
  --workspace dogfood \
  --from 2026-03-08 --to 2026-03-08 \
  --timezone America/New_York --json
/private/tmp/traceary-issue-1432 list \
  --db-path /private/tmp/traceary-issue-1432-qa.db \
  --workspace dogfood \
  --from 2026-03-08 --to 2026-03-08 \
  --timezone America/New_York --json
/private/tmp/traceary-issue-1432 search absent \
  --db-path /private/tmp/traceary-issue-1432-qa.db \
  --workspace dogfood \
  --from 2026-03-08 --to 2026-03-08 \
  --timezone America/New_York --json
```

## 結果

- report は `requested_to=2026-03-08` を保持しました。
- 実効期間はニューヨークの夏時間開始をまたぐ 23 時間の
  `[2026-03-08T05:00:00Z, 2026-03-09T04:00:00Z)` になりました。
- テキスト出力の主な `Period` 行は要求した暦日の終了を表示し、実効上限は
  排他境界として別の行に表示しました。
- `list` と `search` は新規ストアで同じ日付・タイムゾーン入力を受け付け、
  期間規則を変えずに空の JSON 配列を返しました。
- `--timezone Mars/Olympus` はクエリ実行前に終了コード 1 と明示的な
  不正タイムゾーンエラーで拒否されました。
- 単体テストでは、夏時間終了日の 25 時間、RFC3339 の正確な排他時刻、
  上限省略時のスナップショット、CLI/MCP の同一 criteria も固定しています。
