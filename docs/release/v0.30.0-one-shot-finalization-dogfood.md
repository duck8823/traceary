## Deterministic process outcomes

The wrapper supervised four shell processes with explicit session IDs:

```sh
traceary session run --db-path /private/tmp/traceary-1436.db --session-id dogfood-success -- sh -c 'printf success-output'
traceary session run --db-path /private/tmp/traceary-1436.db --session-id dogfood-failure -- sh -c 'exit 7'
traceary session run --db-path /private/tmp/traceary-1436.db --session-id dogfood-timeout --timeout 20ms -- sh -c 'sleep 1'
traceary session run --db-path /private/tmp/traceary-1436.db --session-id dogfood-signal -- sh -c 'kill -TERM $$'
```

Observed wrapper exit codes were `0`, `7`, `124`, and `143`. SQLite inspection showed `runtime_mode=one_shot`, terminal reasons `success`, `failure`, `timeout`, and `signal`, exactly one `session_ended` event per session, and `PRAGMA integrity_check=ok`. An in-process adapter test separately forces a writer failure and verifies `aborted_stream` with exit code `74`.

## Codex exec integration

The built Traceary binary was placed first on `PATH`, then used to supervise a read-only, ephemeral Codex invocation from an empty temporary workspace:

```sh
NO_BROWSER=1 traceary session run \
  --db-path /private/tmp/traceary-codex-dogfood.db \
  --session-id dogfood-codex-exec \
  --agent codex \
  --workspace /private/tmp/traceary-codex-dogfood \
  -- codex exec --ephemeral --skip-git-repo-check --sandbox read-only \
  -C /private/tmp/traceary-codex-dogfood \
  -o /private/tmp/traceary-codex-last.txt \
  'Reply exactly PASS'
```

- Prompt SHA-256: `3ea9dc67727f144161bbf12b01968e0db4aee65ea7ae4e6d9d66fa17c4860601`
- Output: `PASS`
- One session row: `dogfood-codex-exec`, `one_shot`, `success`
- All hook events used the wrapper session ID; no host-ID session row was created.
- Codex `Stop` produced transcript boundaries, while the wrapper alone produced the single `session_ended` event.
- Store integrity: `ok`

The first sandboxed transport attempt failed before model execution because the local in-process app-server could not initialize. One authenticated host-transport retry succeeded. No repository source, credentials, token values, browser state, or production data was sent.

## Decision

The one-shot wrapper is acceptable for the v0.30.0 sequence. It derives completion only from the supervised process, preserves interactive `Stop` behavior, correlates nested Codex hooks to the wrapper identity, and persists one authoritative terminal transition.
