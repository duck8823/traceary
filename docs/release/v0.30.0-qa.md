# v0.30.0 release QA and dogfood

[日本語](./v0.30.0-qa.ja.md)

## Scope

The release candidate was exercised on a fresh disposable SQLite store in addition to the feature-specific copied-store and concurrency dogfood runs. The final pass covered metadata-only reads, workspace and delivery identity diagnostics, one-shot lifecycle outcomes, full and partial reports, date-only intervals, legacy wide rendering, and invalid resource controls.

## Final fresh-store pass

The candidate binary was built from the final implementation head and used a store under `/private/tmp`.

- `session run` recorded success (exit 0), failure (exit 7), and timeout (exit 124) as ended one-shot sessions with terminal reasons `success`, `failure`, and `timeout`.
- `session list --json` returned all three sessions with two lifecycle events each.
- `list --wide` emitted the eight-field legacy header, while `list --fields ... --json` returned only the requested projection.
- A complete report with `--page-size 2` scanned all 3 sessions and 6 events. A partial report with `--result-cap 1` reported `coverage=partial`, per-source observed ranges, `response_truncated=true`, and `truncation_reason=result_cap` without presenting incomplete ratios.
- Date-only `--from 2026-07-22 --to 2026-07-22 --timezone UTC` resolved to the half-open interval `[2026-07-22T00:00:00Z, 2026-07-23T00:00:00Z)`.
- `report workspace-identity --json` reported complete observation coverage, no conflicts, and no duplicate delivery attempts for the fresh store.
- `--page-size 9223372036854775807` exited 1 with `page size must be less than or equal to 100000`; no panic or runtime stack was emitted.

## Follow-up finding and disposition

The release-wide pass found that a huge report page size panicked while preallocating a slice. Follow-up Issue [#1479](https://github.com/duck8823/traceary/issues/1479) was filed, implemented in PR [#1480](https://github.com/duck8823/traceary/pull/1480), independently re-reviewed after a blocking test-coverage finding, and merged only after full local validation and CI passed. The fixed candidate produced a bounded validation error in the repeated dogfood command.

## Release decision

Pass. Every release-blocking QA finding is closed and verified. No unresolved data-loss, compatibility, security, or availability blocker remains for v0.30.0.
