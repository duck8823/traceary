# v0.30.0 command-audit normalization dogfood

[日本語](./v0.30.0-command-audit-normalization-dogfood.ja.md)

Date: 2026-07-22  
Issue: #1440

## Scope

A fresh temporary store was used. No production or user database was read or changed.

The exercise covered direct and verified `rtk`-wrapped identity, exit-code success with quoted failure text, structured failure reasons, report aggregation, encrypted bundle round-trip, and SQLite integrity.

## Results

- `rtk git status` persisted as `wrapper=rtk`, `command_name=git`.
- Direct `git status` persisted as `wrapper=''`, `command_name=git`.
- The two successful `git` rows aggregated together with `failed_count=0`, despite quoted failure text.
- Structured failure totals were exactly `timeout=3`, `hook_denied=1`, `exit_code=1`.
- The three timeout rows produced one `go` failure loop with count 3.
- Bundle import restored all 7 events and all 7 command-audit rows with the same normalized identity and outcome fields.
- `PRAGMA integrity_check` returned `ok` in both fresh databases.

## Follow-up result

No additional defect was found in this focused exercise. The release-wide QA pass still runs before the v0.30.0 tag and files follow-up issues for release-level defects.
