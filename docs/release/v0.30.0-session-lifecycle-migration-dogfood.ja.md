# v0.30.0 session lifecycle migration dogfood

[English](./v0.30.0-session-lifecycle-migration-dogfood.md)

日付: 2026-07-22

Issue: #1430

## 方法

v0.30 workspace-report QA で作成済みの 3.06 GiB のローカルストアコピーを、APFS の copy-on-write で複製して使いました。Issue #1430 のバイナリが migration 24 を適用したのは、この破棄可能な複製だけです。稼働中のストアと、元になった QA コピーは変更していません。

変更前後は集計件数と `PRAGMA integrity_check` だけを確認しました。イベント本文、session identifier、credential、生の行はこの文書に記録せず、外部レビュアーにも共有していません。別の小さな破棄可能ストアで、バイナリのロールバック互換性も確認しました。現行バイナリで schema 24 を作成し、インストール済み v0.29.0 で session の開始・終了を記録した後、Issue #1430 のバイナリがその終了済み集約を復元し、ラベルを更新できることを確認しました。

## コピーしたストアの母集団

- database size: 3.06 GiB
- event: 212,023 件
- session: 21,228 件
- active session: 161 件
- ended session: 21,067 件
- 開始時 schema: migration 23 個

## Migration の測定

Command: `/usr/bin/time -l traceary-under-test store init --db-path <disposable-copy>`

- 経過時間: trigger を含む最終 migration で 19.51 秒（直前の完全測定では 19.63 秒）
- 最大常駐メモリ: 36,782,080 bytes (35.1 MiB)
- migration 後 schema: 24 個
- runtime mode: 21,228 件すべて `interactive`
- terminal reason: active 161 件は空、ended 21,067 件は `legacy_unknown`
- active なのに reason がある不正行: 0 件
- ended なのに reason がない不正行: 0 件
- migration 後の event: 212,023 件
- `PRAGMA integrity_check`: `ok`
- active session の terminal guard: insert/update trigger が存在し、active-with-reason の直接 insert を拒否

経過時間は、このローカル APFS コピーでの一度限りの migration 測定であり、一般的な性能保証ではありません。migration は追加型で、更新するのは 2 つの lifecycle column だけです。既存の集約フィールドが保持されることは自動テストでも確認しています。

## ロールバック互換性

インストール済み `traceary 0.29.0` は、最終 schema 24 の破棄可能データベースで session を正常に開始・終了できました。新しい 2 column には安全な default があり、guard は終了済み行の空 reason を許可するためです。guard が拒否するのは、active 行に terminal reason を付けた状態だけです。旧バイナリが記録した終了では、想定どおり `terminal_reason` は空でした。Issue #1430 のバイナリはその行を安全側の `legacy_unknown` として復元し、終了マーカーを変更せずにラベル更新を完了しました。破棄可能データベースの整合性も維持されました。

## リリース判断

この lifecycle migration は v0.30.0 の実装を進める条件を満たします。履歴 session を one-shot と推測せず、履歴の終了理由を作り上げず、元データと件数を保持しています。インストール済み v0.29.0 へバイナリを戻しても、追加型 schema は書き込み不能になりません。
