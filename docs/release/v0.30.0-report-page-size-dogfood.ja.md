# v0.30.0 レポートページサイズ dogfooding

[English](./v0.30.0-report-page-size-dogfood.md)

## 発見事項

v0.30.0 のリリース全体 QA で、空の使い捨てストアに対して `--page-size 9223372036854775807` を指定して `report` を実行しました。修正前のバイナリは `loadCappedReportRows` の `runtime error: makeslice: cap out of range` で panic し、終了コード 2 になりました。リリースを止める可用性不具合として、フォローアップ Issue #1479 を作成しました。

## 修正後バイナリの検証

Issue #1479 のバイナリで同じコマンドを再実行しました。

```sh
traceary report \
  --db-path /private/tmp/traceary-v030-qa.db \
  --from 2026-07-01 \
  --to 2026-07-01 \
  --page-size 9223372036854775807 \
  --json
```

修正後は `page size must be less than or equal to 100000` という通常の検証エラーで終了コード 1 になり、panic や runtime stack は出ませんでした。単体・CLI・MCP テストでも、許容最大値、最大値 + 1、巨大整数を確認しています。

## 判定

合格。CLI と MCP は内部ページサイズの最大値 100,000 を共有します。不正なリソース制御値はレポート query 実行前に失敗し、通常の完全・部分集計の挙動は変わりません。
