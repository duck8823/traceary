# v0.30.0 完結型セッション修復の dogfood

[English](./v0.30.0-one-shot-repair-dogfood.md)

## 対象と安全境界

- Issue: #1434
- バイナリ: `/private/tmp/traceary-1434`
- コピーしたストア: `/private/tmp/traceary-1434-backlog.db`
- 証拠マニフェスト: `/private/tmp/traceary-1434-backlog-evidence.json`
- マニフェストの SHA-256: `67803a7db694cf83b34aac1c71f3319592327119988b1f7b5be13ed76937a343`
- 候補: 10,170 件
- 実運用ストアは変更していない。
- 生のセッション ID と証拠マニフェストはリポジトリへ追加していない。

現在のストアでは、元のバッチ処理後に stale session の GC が実行されていたため、観測済みの backlog 10,170 件はすでに `legacy_unknown` で終了していた。SQLite の整合したバックアップを `/private/tmp` に作成し、既存の `ended_at` を保守的なオペレーター確認済みプロセス完了時刻としてマニフェストへ保持した。その上で、コピー内の候補行だけを GC 前に観測した active 状態へ戻した。これにより、実運用データを変更せず、transcript 本文、idle 時間、workspace 所属から完結型を推測することもなく、元へ戻せる修復リハーサルを構成した。

## Dry-run

dry-run は明示された 10,170 件すべてを `eligible` と判定し、変更件数は 0 件だった。コピー内の対象は active、interactive の 10,170 件のままで、`one_shot_repair` event も 0 件だった。コマンド実行前後のデータベース SHA-256 は同一だった（`72f5222376c911c0072c4f59c24a3a6c2bcc5c6063c685584a2bfada598e88a0`）。ストア全体の集計は次のとおりだった。

| 指標 | 実行前 | 実行後 |
|---|---:|---:|
| Active | 10,335 | 10,335 |
| Stale | 10,181 | 10,181 |
| 正常終了 | 0 | 0 |
| 型付き異常終了 | 0 | 0 |

## 適用と冪等性

適用前に必須の `/private/tmp/traceary-1434-backlog-before-apply-final.db` を作成し、その後で原子的な修復トランザクションを開始した。10,170 件すべてが適用された。

| 指標 | 実行前 | 実行後 |
|---|---:|---:|
| Active | 10,335 | 165 |
| Stale | 10,181 | 11 |
| 正常終了 | 0 | 10,170 |
| 型付き異常終了 | 0 | 0 |

SQLite を検査し、対象 10,170 行が `runtime_mode=one_shot`、`terminal_reason=success` になり、`one_shot_repair` が記録した相異なる 10,170 件の `session_ended` event が 1 セッションにつき 1 件だけ存在することを確認した。対象外 11,082 セッションの集計値は変化せず、`PRAGMA integrity_check` は `ok` だった。

2 回目の適用でも必須の適用前バックアップを作成した。10,170 件すべてが `already_terminal` と判定され、変更は 0 件、修復 event は 10,170 件のままだった。ストア整合性も `ok` を維持した。

## ロールバックのリハーサル

必須の適用前バックアップを別のデータベースパスへ復元した。復元後の対象は active、interactive の 10,170 件、修復 event は 0 件で、対象外の集計値も同一だった。`PRAGMA integrity_check=ok` も確認した。

## 判定

証拠に基づく修復を v0.30.0 の次工程へ進めてよい。既定では書き込みを行わない dry-run であり、適用前バックアップの成功を必須とし、指定された全候補の判定理由を出力する。対象となる遷移は 1 トランザクションで反映し、マニフェストにないセッションを変更せず、再実行しても重複せず、復元経路も検証できている。
