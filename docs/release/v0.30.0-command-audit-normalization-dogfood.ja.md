# v0.30.0 command audit 正規化の dogfooding

[English](./v0.30.0-command-audit-normalization-dogfood.md)

実施日: 2026-07-22  
Issue: #1440

## 対象

新規の一時ストアだけを使い、本番 DB やユーザー DB は参照・変更していません。

直接実行と確認済みの `rtk` wrapper 経由の識別、引用された失敗文言を含む終了コード 0 の成功、構造化された失敗理由、report 集計、暗号化 bundle の往復、SQLite の整合性を確認しました。

## 結果

- `rtk git status` は `wrapper=rtk`、`command_name=git` として保存されました。
- 直接実行の `git status` は `wrapper=''`、`command_name=git` として保存されました。
- 成功した 2 件の `git` は同じ行へ集計され、引用された失敗文言があっても `failed_count=0` でした。
- 構造化された失敗理由は `timeout=3`、`hook_denied=1`、`exit_code=1` と一致しました。
- timeout 3 件は、count 3 の `go` 失敗ループ 1 件として検出されました。
- bundle import は 7 events と 7 command-audit rows をすべて復元し、正規化済み識別子と実行結果フィールドが一致しました。
- 両方の新規 DB で `PRAGMA integrity_check` は `ok` でした。

## フォローアップ結果

この限定的な検証では追加の不具合は見つかりませんでした。v0.30.0 tag 前には別途リリース全体の QA を実施し、リリース水準の不具合が見つかった場合はフォローアップ Issue を作成します。
