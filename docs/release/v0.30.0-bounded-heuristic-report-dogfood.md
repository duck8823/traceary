# v0.30.0 bounded heuristic report dogfood

[日本語](./v0.30.0-bounded-heuristic-report-dogfood.ja.md)

Date: 2026-07-22

Issue: #1467

## Method

The installed v0.29.0 binary created a consistent backup of the active local store at `/private/tmp/traceary-v030-1467-dogfood.db`. The Issue #1467 binary migrated only that copy. No live row was changed. The before/after binaries then ran `report workspace-identity --json` against the same 3.06 GiB copy under `/usr/bin/time -l`.

The optimized binary was measured twice:

1. default body-free report, with no heuristic request;
2. explicit `--include-heuristic --heuristic-limit 5000` bounded measurement.

Only aggregate JSON fields, elapsed time, and peak resident memory are recorded here. No event body, raw row, credential, or identifier sample was committed or sent to an external reviewer.

## Copied-store population

- events: 212,023
- eligible historical hook prompt/transcript rows: 45,150
- existing dedupe archive rows: 0
- runtime delivery attempts: 0 (the active source store predates the v0.30 ledger)

## Results

| pre-fix default (`main` at `a948f4ac`) | implicit complete | 45,150 | 398 | 22.06 s | 733,560,832 bytes (699.6 MiB) |
| #1467 default | `not_requested` | 0 | 0 | 0.01 s | 30,752,768 bytes (29.3 MiB) |
| #1467 explicit limit 5,000 | `partial` | 5,000 / 45,150 | 172 | 0.59 s | 179,486,720 bytes (171.2 MiB) |

The default path is 2,206 times faster in this warm-cache comparison and uses about 4.2% of the pre-fix peak RSS. These ratios describe this local copied-store run only; the absolute numbers depend on filesystem cache and body sizes.

The coverage query also changed from a correlated scan of all events and workspace observations to an index-driven observation-to-event join. Before that query correction, a cold copied-store attempt spent 81.41 seconds before failing later on an intentionally stale development schema. That failed run is diagnostic evidence only and is not used in the comparison table.

## State and safety checks

- Default JSON reported `measurement_state=not_requested` and still returned exact-delivery/workspace metrics.
- Explicit bounded JSON reported `measurement_state=partial`, `eligible_count=45150`, and `scanned_count=5000`.
- CLI tests cover all four states: `not_requested`, `partial`, `complete`, and `failed`.
- The failed-state test proves exact delivery fields are emitted before the command returns the heuristic error.
- The store-management use case rejects `Apply=true` together with a scan bound.
- Existing store dedupe dry-run/apply/restore tests remain unchanged and pass.
- After the report runs, the copied store still contained 212,023 events and zero dedupe archive rows.

## Release decision

The #1467 blocker is satisfied. Routine workspace-identity self-inspection no longer materializes bodies, explicit historical measurement is bounded and stateful, exact delivery remains independent, and the copied large-store latency/memory evidence is acceptable for v0.30.0.
