Memory substrate for AI coding agents.

Traceary records session lifecycle, shell command audits, and — where the host hooks expose them — prompts and compact summaries from Claude Code and Codex (plus legacy Gemini CLI compatibility) into a single local SQLite store, so context survives /clear, /compact, and the gaps between agents.

The problem

AI-assisted development gets messy quickly.

Git history tells you what changed, but not why. Sessions disappear after compaction. Output scatters across three different agents. Traceary keeps the timeline whole.

Context lost on /clear

Agent context disappears after compaction. Decisions and intent vanish with it.

Git shows what, not why

Commit logs miss the reasoning, the failed attempts, and the decisions you reverted.

Audits float free

Hard to connect a shell command back to the agent, session, and prompt that produced it.

Multiple agents, scattered logs

Claude Code and Codex each keep their own state. Cross-tool history needs one home — and new hosts like Antigravity will add to the fragmentation.

Worktrees & parallel sessions

Multiple sessions and worktree moves break the timeline you're trying to follow.

No place for durable facts

Decisions and constraints worth keeping around have no home outside the session log.

Three-layer model · v0.5+

More than a log. A memory substrate.

Traceary organizes records the way agent workflows actually need them — raw timeline at the bottom, working memory in the middle, durable facts at the top.

LAYER 01
// audit · archive

Source-of-truth timeline

Raw events — prompts, transcripts, command audits, session boundaries — kept untouched for inspection, search, and forensic review.

  • prompt
  • command_executed
  • session_started / ended
  • transcript
LAYER 02
// working memory

Curated context packs

Handoff packs assembled from recent sessions, so the next agent — or a resumed session — starts with the right context, not the whole log.

  • compact_summary
  • session handoff
  • workspace breakdown
  • activity rollups
LAYER 03
// durable memory

Facts that should survive

Decisions, constraints, preferences, and artifact refs — the small set of things that need to outlive any single session and be retrieved on demand.

  • decision
  • constraint
  • preference
  • lesson
  • artifact
Getting started

Three steps. Then it just records.

The CLI is the foundation — agent plugins invoke the traceary binary via hooks. Install once, then plug in whichever agents you use.

STEP 01

Install the CLI

Required first. macOS / Linux archives on GitHub Releases too.

# Homebrew (recommended)
brew tap duck8823/traceary https://github.com/duck8823/traceary
brew install traceary

# or via Go
go install github.com/duck8823/traceary@latest
STEP 02

Wire your agent

Active packages for Claude Code, Codex, and Antigravity; legacy Gemini CLI compatibility.

STEP 03

Verify and start

Then open a session and start recording.

traceary doctor

# start a session
sid=$(traceary session start --client dogfood --agent codex)
traceary log --client dogfood --agent codex --session-id "$sid" "Investigating failing tests"
traceary search boom --json
Host capture matrix

One query surface. Two active capture tiers.

Once installed, every host shares the same CLI and MCP commands. What differs is how much each one captures automatically through hooks.

Host Session lifecycle Tool audit Prompt Compact summary Tier
Claude Code Full Bash + MCP + failure ● ● FULL
Codex SessionStart + Stop Tool hooks ● — PARTIAL
Gemini CLI (legacy) SessionStart + End Tool hooks — — BASIC
Antigravity PreInvocation + Stop run_command hooks — — PARTIAL

Gemini CLI details are legacy compatibility for existing installs. Antigravity uses the documented public hook/plugin surface only.
